# GitHub 部署说明

## 第一步：上传文件

把压缩包解压后的所有文件上传到 GitHub 仓库根目录。

重点检查以下文件必须存在：

```text
.github/workflows/update.yml
public/index.html
public/app.js
public/styles.css
scripts/update.js
scripts/sources.js
package.json
```

## 第二步：开启 GitHub Pages

进入仓库：

```text
Settings → Pages → Build and deployment → Source → GitHub Actions
```

## 第三步：运行一次更新

进入仓库：

```text
Actions → Update Entertainment Rank → Run workflow
```

运行成功后，页面会自动发布。

## 第四步：确认自动更新

工作流里已经写好每小时自动更新：

```yaml
schedule:
  - cron: '8 * * * *'
```

## 常见问题

### 1. 页面打开是 404

先检查：

- `Settings → Pages` 是否选择了 `GitHub Actions`
- `Actions` 里最近一次工作流是否成功
- 仓库默认分支是不是 `main`

### 2. Actions 失败，提示某个来源 HTTP 403 / 503

这是 TopHub 对请求有限制或临时不可用。先重新运行一次；如果连续失败，可以配置仓库 Secret：

```text
TOPHUB_COOKIE
```

### 3. 为什么不用旧版本 Actions

本项目 workflow 使用的是：

```yaml
actions/checkout@v6
actions/setup-node@v6
actions/configure-pages@v6
actions/upload-pages-artifact@v5
actions/deploy-pages@v5
```

没有使用 `checkout@v3`、`setup-node@v3` 或其他老版本。
